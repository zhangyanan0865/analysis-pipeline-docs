Extracted Chain
从 ODB 提取 load 和 stress 原始数据 · 主链第一层
这条链做什么

从 Abaqus ODB 文件中提取当前主链所需的原始数据，写入标准化的 extracted/ 目录，供下游所有链条消费。这是整条主链的起点，上游只有 ODB，不接收任何 legacy 中间结果。

执行流程
1
配置提取范围
在两张配置表里登记要提取的 case 和 frame，再跑脚本。
先改配置
2
提取 Load（UF 曲线）
extract_load_from_odb.py
从 ODB 提取每个 case 的 UF 曲线，写入 extracted/load/。
3
提取 Stress（界面应力曲线）
extract_stress_from_odb.py
按 target_frames 配置，从 ODB 提取每帧的 stress 曲线，写入 extracted/stress/。
python scripts/extracted/extract_load_from_odb.py
python scripts/extracted/extract_stress_from_odb.py
两个脚本可以独立运行，也可以顺序跑。load 提取要先于 interpolation 跑，因为 prepare_interpolation_cases.py 会检查 extracted/load 是否存在。
配置表

config/extracted/load_cases/extract_load_cases.csv

控制哪些 case 参与 load 提取。

字段说明必填
enabled0 = 跳过，1 = 提取必填
case_name对应 ODB 文件的 case 标识必填

config/extracted/target_frames/target_frames_by_column.csv

控制每个 case 需要提取哪些 stress frame。

字段说明必填
case_namecase 标识必填
frame_ids需要提取的帧号列表必填

职责边界

本链只负责提取和标准化输出。不负责：数据清洗、帧间插值、曲线平滑、下游分析、结果展示层。

提取层脚本只写入 extracted/，不直接写入 analysis/。
